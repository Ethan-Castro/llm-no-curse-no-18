Training Pipeline

1

Corpus Collection

Stream kid-safe educational text from two curated sources.

Cosmopedia (70%) — HuggingFace's 25B-token synthetic educational dataset, filtered by audience field to only include: young_children, grade_school_students, middle_school_students. Subsets: stories, auto_math_text, openstax, stanford, khanacademy, wikihow. Adult-audience rows and web_samples are excluded entirely.

Project Gutenberg (30%) — Public domain children's literature via the Gutendex API. English-only, topic=children. Boilerplate headers/footers stripped via regex.
2

Content Safety Audit

Every single document passes through a multi-layer safety filter. If any layer flags it, it's removed.

Pass 0 — Hardcoded Blocklist: 200+ terms covering English profanity, Spanish/French/German/Italian/Portuguese curse words, racial slurs, sexual terms, leetspeak variants (f@ck, sh1t, b!tch), and archaic offensive language common in old literature. Case-insensitive regex matching. Instant, runs on CPU.

Pass 1 — better_profanity: Python library with its own comprehensive built-in word list. Catches word-level profanity the blocklist might miss. Runs locally, zero cost.

Pass 2 — Llama Guard 4 12B: Meta's purpose-built content safety classifier (fine-tuned from Llama 4 Scout). Understands context, not just words. Classifies against the MLCommons hazard taxonomy: violence, sexual content, hate speech, self-harm, criminal activity, and more. Each document sent via API and must return exactly safe to pass. 200 concurrent API calls via OpenRouter for speed.
3

Parquet Sharding

Clean documents are written to nanochat-compatible Parquet shards.

Format: text column, 1024-row row groups, zstd level 3 compression, ~250M chars per shard. Last shard alphabetically = validation shard (nanochat convention). Padded to exact multiples of 1024 rows.
4

Synthetic Dialogue Generation

5,000 tutoring conversations generated by Gemma 4 31B, then safety-checked by Llama Guard 4.

Generator: Google Gemma 4 31B IT via OpenRouter. 95 topic templates across math, science, animals, history, nature, how-things-work, life skills, and space. 50 variations per topic. Temperature 0.9 for diversity.

Format: Each dialogue is 4-8 turns between a curious child (age 8-12) and a patient, encouraging tutor. Output as JSONL arrays matching nanochat's CustomJSON SFT format.

Safety: Every generated dialogue passes through: (1) hardcoded blocklist, (2) Llama Guard 4 context check. Rejected if either flags it. 50 concurrent generation + 30 concurrent safety checks.
5

Tokenizer Training

Train a BPE tokenizer from scratch on our clean corpus.

Uses nanochat's tok_train.py — trains a byte-pair encoding tokenizer directly on the kid-safe corpus. This means the tokenizer's vocabulary is optimized for educational children's content, not general web text.
6

Pretraining

Train the base language model on clean Parquet shards using nanochat.

Framework: Karpathy's nanochat — a from-scratch GPT training codebase with modern architecture features (relu² activation, smear gate, value embeddings, x0-parameterization).

Model sizes: --depth=20 for ~300M params, --depth=30 for ~700M params.

Hardware: 4x RTX 4090 GPUs rented via Vast.ai or RunPod (~$1.20-1.50/hr). Estimated training time: 70-90 hours. Budget: $85-135.
7

Supervised Fine-Tuning (SFT)

Fine-tune on the 5,000 tutoring dialogues to teach conversational behavior.

Uses nanochat's chat_sft script. The model learns to be a helpful, encouraging tutor that explains concepts at a child's level. Input format: CustomJSON JSONL with alternating user/assistant turns.
8

Local Deployment

Convert to GGUF for local inference on consumer hardware.

Native inference: nanochat's built-in chat_cli.py — full quality, uses all architecture features.

GGUF conversion: Lossy v1 — maps weights to LLaMA-like structure. Drops nanochat-specific features (smear gate, value embeddings). Enables Q4_K_M quantization for fast CPU inference via llama.cpp or Ollama. Best-effort quality.

Safety Architecture

Four independent layers. A document must pass ALL of them to make it into training data.

Layer 0 — Source Selection

Pre-filtered Sources

Only kid-safe Cosmopedia subsets (filtered by audience metadata) and Gutenberg children's literature. No web scraping. No Reddit. No Wikipedia. No Common Crawl.

Layer 1 — Hardcoded Blocklist

200+ Banned Terms

English profanity + Spanish, French, German, Italian, Portuguese curse words. Racial slurs. Sexual terms. Leetspeak variants (f@ck, sh1t, a$$). Archaic offensive terms from old literature. Case-insensitive regex.

Layer 2 — Profanity Detection

better_profanity Library

Local Python library with comprehensive built-in word lists. Catches profanity variants and misspellings the blocklist might miss. Zero cost, instant execution.

Layer 3 — Llama Guard 4 12B

Contextual AI Safety

Meta's content safety classifier. Understands meaning, not just words. Catches: violence, sexual content, hate speech, self-harm, dangerous activities, and more. MLCommons hazard taxonomy. 200 concurrent API checks.
